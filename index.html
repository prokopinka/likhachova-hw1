• git config --global user.name blabla - задать имя юзера git
  • config --global user.email blabal@bla - задать почту пользователя git
  • init - инициализировать репозиторий
  • git status - узнать в каком состоянии находится рабочая папка (ее stage этап)
  • git add - добавить файл в стадию stage (проиндексировать /определить)
  • git commit - внести изменения в репозиторий (отгрузить свои файлы/ изменения)
  • git log - узнать информацию о коммитах/изменених/мерджах в репозитории
  • git checkout - перключиться на другую версию/ ветвь
  • git branch - определить/ перечислить /удалить существующие ветви, /переключиться на другую ветвь
  • git push - отправить коммит в удаленный репозиторий
  • git pull - стянуть изменения с удаленного репозитория
  • git merge - соединить ветви
  • git clone -  создать копию существующего репозитория

  • .gitignore файл - используется чтобы исключить определенные файлы/ от автоматического добавления в репозиторий и их отслеживания (зачастую такие файлы, которые автоматически генерируют какие-либо данные/ логи)

  • git revert - совершить коммит, который отменяет предыдущий коммит / откатывается до старой версии (условно, с плохими/ неправильными изменениями). В истории будет выглядить так, буд-то коммита с плохими изменениями и вовсе не было
  • git reset - откатится до какой-либо старой версии /коммита, стирая все данные (есть три аргумента для разных вариантов удаления --hard, --soft, --mixed))
  • git rebase - переорганизовать струкутуру (комимиты) репозитория
  • git cherry-pick - забрать несколько коммитов с одной ветки, без полного сливания с этой веткой
  • git stash - сохранить незакоммиченные локальные изменения
